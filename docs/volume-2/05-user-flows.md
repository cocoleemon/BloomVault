# 🌸 User Flows

> *"A great flow feels so natural that users never stop to think about the next step."*

---

# Experience Goal

Design task flows that are intuitive, efficient, and predictable.

Every interaction should require as little effort as possible while preserving the thoughtful and personal nature of BloomVault.

Users should always know what to do next.

---

# Introduction

User flows describe the exact sequence of actions users take to complete specific tasks within BloomVault.

Unlike user journeys, which focus on motivations and emotions, user flows focus on interactions and decision points.

These flows define the expected behavior of the product and serve as a reference for design and development.

---

# Flow 1 — First-Time Onboarding

### Goal

Introduce BloomVault and prepare the user's personal beauty library.

```text
Open App
      ↓
Welcome
      ↓
Introduction
      ↓
Create Account / Sign In
      ↓
Complete Basic Profile
      ↓
(Optional) Skin Profile
      ↓
Enter Library
```

**Success**

The user understands BloomVault's purpose and arrives in an empty but welcoming personal library.

---

# Flow 2 — Sign In

# Flow Principles

Every user flow should:

* Minimize unnecessary steps.
* Keep primary actions visible.
* Offer clear feedback after important actions.
* Preserve progress whenever possible.
* Allow users to easily return to where they were.

---

# Error Recovery

When users encounter an issue, BloomVault should:

* Explain what happened in plain language.
* Suggest the next best action.
* Avoid losing user input.
* Make recovery simple and reassuring.

Errors should feel like temporary interruptions—not dead ends.

---

# Success Criteria

A successful user flow is one where:

* The goal is completed with minimal effort.
* Users feel confident throughout the process.
* Navigation feels predictable.
* Progress is never lost.
* The experience encourages users to continue exploring.
